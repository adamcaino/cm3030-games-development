Game Idea Summaries

**Adam:** A 3D top-down game with camera angles similar to Diablo or The Sims. Point and click movement of the player. The player must navigate a lab they are locked in to solve various puzzles leading to the player creating the cure for a zombie virus they have been bitten with. The player has 10 minutes to solve the puzzles before they die. World interactions include investigating various items (vials, books, computers, shelves, etc.) for clues to create the cure. There may be randomisation, whereby each run the solution is slightly different (e.g., a blue vial is helpful one run and the next run it is damaging, access codes are random, etc.).

**Kristin:** A 3D first-person shooter (FPS) game similar to Portal. The player has a gun which allows the player to manipulate the world using physics-based interactions; these help create solutions to puzzles such as hitting switches and moving objects around to create paths to an exit or the next area of the game. There may also be an AI companion you can use to help solve puzzles. The world may be procedurally generated each time.

**Mirza:** Rogue-like/lite top-down game (likely 2D). The player moves characters using a tile-based system and keyboard controls. The player controls four characters at once, each character is of a different RPG class (such as tank, DPS, support, flexi). These main classes have sub-classes which are randomised each run (e.g., if the character is a Mage, they may be a Frost mage one run, and a Necromancer in the next run). There would be a stats system, where each run each character's stats are randomised as well (e.g., their strength is 12 one run, and 10 the next). The dungeon itself would be fixed / it wouldn't change, as the game is focusing on a "death loop" rather than a full rogue-like with completely random worlds each time. There may be a turn-based battle system and puzzles to solve within the dungeon to progress.

**Andi:** A combination of all ideas, or a simple game similar to chess, 9ball, or Heroes of Might and Magic. Andi prefers 3D games as it aligns with our course. Andi is familiar with 3D applications such as AutoCAD and 3D Studio Max.
